# antnusnv.com — портфолио

Статический сайт-портфолио на [Astro](https://astro.build). Собирается и публикуется на GitHub Pages автоматически через GitHub Actions при пуше в `main`.

## Стек

- **Astro** — статическая генерация, файловая маршрутизация страниц
- Чистый CSS с custom properties под токены дизайн-системы (`src/styles/tokens.css`)
- Небольшой vanilla JS для интерактивных блоков (слайдер до/после, переключатель языка)

## Локальный запуск

Требуется Node.js 22+.

```bash
npm install
npm run dev
```

Сайт откроется на `http://localhost:4321`.

Полезные команды:

```bash
npm run build    # собрать статику в dist/
npm run preview  # локально просмотреть собранную версию из dist/
```

## Структура проекта

```
src/
├── components/   # переиспользуемые блоки (Header, Footer, Button, карточки кейсов и т.д.)
├── layouts/      # общая обёртка страниц (BaseLayout, CaseLayout)
├── pages/        # маршруты сайта: index.astro = /, pages/cases/*.astro = /cases/*
├── styles/       # токены дизайн-системы, reset, глобальные стили
├── i18n/         # RU/EN словари для переключателя языка
└── data/         # контент карточек кейсов на главной
public/
├── CNAME         # домен для GitHub Pages
└── images/       # изображения кейсов, фото, логотипы
```

## Деплой

Деплой полностью автоматический: пуш в ветку `main` → GitHub Actions (`.github/workflows/deploy.yml`) собирает проект и публикует `dist/` на GitHub Pages. Вручную ничего собирать и заливать не нужно.

Чтобы задеплоить руками (например, без пуша) — во вкладке **Actions** репозитория запустить workflow `Deploy to GitHub Pages` кнопкой *Run workflow*.

### Первый деплой — настройка на стороне GitHub

1. Запушить репозиторий на GitHub (ветка `main`)
2. В репозитории: **Settings → Pages → Source** выбрать **GitHub Actions**
3. Дождаться, пока отработает workflow во вкладке **Actions** — после первого успешного запуска сайт будет доступен на `https://<username>.github.io/<repo>/` и на подключённом домене (см. ниже)

## Подключение домена antnusnv.com

Домен уже куплен, сейчас указывает на другой хостинг. Чтобы переключить его на GitHub Pages:

1. **В настройках репозитория**: Settings → Pages → Custom domain → ввести `antnusnv.com` → Save. Файл `public/CNAME` с этим доменом уже лежит в репозитории и задеплоится вместе с сайтом, так что GitHub подхватит домен и сам, но явно указать его в настройках всё равно нужно.
2. **В панели регистратора/Cloudflare, где сейчас настроен DNS домена**, заменить существующие записи (сейчас указывающие на Lovable) на записи GitHub Pages:

   - Для корневого домена (`antnusnv.com`) — четыре A-записи на IP GitHub Pages:
     ```
     185.199.108.153
     185.199.109.153
     185.199.110.153
     185.199.111.153
     ```
   - Если нужен `www.antnusnv.com` — CNAME-запись `www` → `<username>.github.io`

   Если используется Cloudflare — на время смены DNS желательно временно выключить проксирование (значок "оранжевого облака" → "серое облако" / DNS only) для этих записей, чтобы GitHub мог выпустить SSL-сертификат.

3. Подождать обновления DNS (обычно от нескольких минут до нескольких часов) и в Settings → Pages убедиться, что появилась галочка **Enforce HTTPS** — значит сертификат выпущен и домен подключён.
4. Старую версию портфолио на Lovable можно отключать/удалять только после того, как `antnusnv.com` уже отдаёт новый сайт — чтобы не остаться без работающего домена в переходный момент.
